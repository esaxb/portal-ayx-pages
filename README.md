# portal-ayx-pages

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的静态仓库。  
本仓库不绑定任何特定域名或网站，仅作为页面文件的集中存放与版本管理场所。  
适合存放各类临时页面、静态展示页、归档页等无需后端支持的 HTML 文件。

## 目录说明

```
portal-ayx-pages/
├── index.html          # 仓库首页（可选）
├── pages/              # 存放独立 HTML 页面
│   ├── example1.html
│   └── example2.html
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有独立 HTML 页面存放于此，每个页面为一个独立的归档单元。  
- `assets/`：页面共用的样式、脚本、图片等资源，便于统一维护。  
- `index.html`：可选的仓库入口页面，用于导航或说明。

## 页面归档说明

- 每个 HTML 页面应自包含或引用 `assets/` 下的公共资源。  
- 页面命名应清晰反映内容或用途，避免使用无意义缩写。  
- 如需引用外部资源（如字体、库），建议使用 CDN 或本地副本，确保长期可用。  
- 页面之间不应耦合，删除或新增单个页面不影响其他页面。

## 维护说明

- 新增页面：直接在 `pages/` 下创建 `.html` 文件，并确保引用资源路径正确。  
- 更新页面：修改对应文件后提交即可，历史版本可通过 Git 追溯。  
- 删除页面：移除文件并清理相关资源引用。  
- 不建议在仓库中存放敏感信息或动态内容。

## 贡献

欢迎提交包含独立 HTML 页面的 Pull Request。  
请确保页面内容合法、无恶意代码，并遵循本仓库的目录结构约定。
